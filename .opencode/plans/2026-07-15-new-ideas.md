# New Ideas: CLT, Memorization Mode, Tab Sync

## Idea 1: CLT Techniques for Hebrew (Communicative Language Teaching)

Research (Holmstedt, Overland, Buth, Kelsey) consistently shows grammar-translation alone underperforms CLT for Biblical Hebrew. Adding vocalization and oral practice would fill the last major pedagogical gap.

### What it would look like

- **Read-aloud prompts** in reading lessons: "Read this verse aloud. Click each word for pronunciation help."
- **Audio-only review mode** (80% exists — `AudioReviewSession` component) — expand to more verses
- **TPR-style drills** for common verbs: "Stand up" (קום), "Sit down" (שב), "Turn" (פנה)
- **Singing the alphabet** — proven most effective for letter/vowel memorization (Kelsey study)
- **Chanting paradigms** — Qal perfect sung to a simple melody

### Effort: Medium (~3-5 hrs)
- Mostly frontend content in lesson explanations + existing AudioReviewSession
- No new infrastructure needed

## Idea 3: Session Persistence & Context Recovery

### Problem
Plans survive context loss (they're markdown files), but task tracking and conversation context don't. If you `/clear` mid-session, you lose track of what you were doing.

### What it would look like
- A `progress.md` update every significant action (already do this!)
- A `/memory-save` command that snapshots current context to a file
- The wiki codebase memory already tracks file changes — could be extended

### Effort: Low (~1 hr)
- Mostly documentation/process — just formalize what we already do
