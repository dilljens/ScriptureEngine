# Oh-My-Agent Improvements Plan

> Integrates `/init-deep`, wiki auto-injection, LSP integration, and skill-embedded MCPs into the existing `/memory*` command workflow.

---

## Architecture: Memory-First Rule Management

All project-level context (AGENTS.md, rules, wiki snapshots) is managed through the `/memory*` command family. The oh-my-agent project registry is the source of truth, and `/memory-init` / `/memory-update` push changes to each project's `.opencode/` files.

```
~/.config/oh-my-agent/projects/         ← Source of truth
  <project-name>.md                      ← Project rules (already done)

Project's .opencode/
  AGENTS.md                              ← Generated from project rules + wiki
  agents/                                 ← Generated by /init-deep
    frontend.md                           ← Focused rules for frontend/
    web.md                                ← Focused rules for web/
    lib.md                                ← Focused rules for lib/
```

## Phase 1: `/init-deep` — Integrated into `/memory*`

### What
Scan the project's source directories and generate focused AGENTS.md files per directory. Each contains:
- **One-liner**: What this directory does
- **Key files**: Most-changed files (from git log)
- **Module-specific rules**: Extracted from the project's oh-my-agent registry

### Integration Points

| Command | New Behavior |
|---------|-------------|
| `/memory-init` | After store init, call `initDeep()` to seed `.opencode/agents/` |
| `/memory-update` | During project sync, call `initDeep()` to regenerate if sources changed |
| `/init-deep` | Standalone: scan project, generate `.opencode/agents/<dir>.md` |

### How `initDeep()` Works

1. Walk source directories: `frontend/src/`, `web/`, `lib/`, `scripts/`, etc.
2. For each directory with ≥3 files, generate an agent file
3. Run `git log --oneline --name-only -100` to find hot files
4. Match directory path against rules in `~/.config/oh-my-agent/projects/<name>.md`
5. Write `.opencode/agents/<slug>.md` for each

### Generated File Example

```markdown
## frontend/src/components/
React components for the learning app.
**Hot files:** ChatPanel.jsx, SearchBar.jsx, LibraryView.jsx
**Rules:** Use refParser.js before writing parsing logic.
            All 9 work IDs in every color/label map.
```

### Loading

The project's `.opencode/opencode.jsonc` loads the agents directory:

```jsonc
{
  "instructions": [".opencode/AGENTS.md", ".opencode/agents/"]
}
```

## Phase 2: Wiki Auto-Injection into AGENTS.md

### What
When `updateAgentsMd()` runs (during `/memory-update`), it reads the project's `docs/wiki/_index.md` and `_standards.md` and injects selected sections into `.opencode/AGENTS.md`.

### What Gets Injected (~450 tokens)

| Source | Section | Tokens |
|--------|---------|--------|
| `_index.md` | `## Architecture` (module list) | ~150 |
| `_standards.md` | All rules and practices | ~200 |
| `_index.md` | `## Quick Reference` (key files) | ~100 |

### What Does NOT

- `_glossary.md` — read on demand
- `features/*.md` — read on demand
- `plans/*.md` — read on demand

## Phase 3: LSP MCP for React/JSX Validation

### What
An MCP server wrapping Language Server Protocol clients. Agent can ask for diagnostics, hover info, and project-wide checks in real time.

### Supported LSP Servers

| Language | Server | Install |
|----------|--------|---------|
| TypeScript/JSX | `typescript-language-server` | `npm i -g typescript-language-server typescript` |
| Tailwind | `@tailwindcss/language-server` | Part of Tailwind CLI |
| Python | `pyright` or `ruff` | `pip install pyright` |

### MCP Server Structure

```
~/.config/oh-my-agent/mcp/lsp-mcp/
  server.py           ← Starts appropriate LSP, exposes tools
  requirements.txt    ← mcp, pygls (or subprocess)
```

### Tools

| Tool | Input | Output |
|------|-------|--------|
| `lsp_diagnostics` | filepath | Errors/warnings for that file |
| `lsp_hover` | filepath, line, col | Type info / docstring |
| `lsp_check_project` | — | Full project diagnostics |

### What It Catches

- `grid-cols-15` → CSS LSP: unknown utility class
- Missing `rehype-raw` → TypeScript LSP: unresolved module
- `r.title` vs `r.verse` → TypeScript LSP: no property

### Project Config (React projects only)

```jsonc
{
  "mcp": {
    "lsp": {
      "type": "local",
      "command": ["python3", "~/.config/oh-my-agent/mcp/lsp-mcp/server.py"],
      "enabled": true
    }
  }
}
```

## Phase 4: Skill-Embedded MCP Servers

### What
Skills can bundle a `server.py` alongside their `SKILL.md`. When loaded via `/skill`, the MCP server registers tools that the agent can call — not just instructions to follow.

### Structure

```
~/.config/oh-my-agent/skills/consistency-check/
  SKILL.md      ← Instructions (loaded on /skill)
  server.py     ← MCP tools (new)
```

### How It Gets Installed

Extend `deployComponents()` in the knowledge plugin:

```python
for skill_dir in os.listdir(skills_path):
    server_path = join(skills_path, skill_dir, "server.py")
    if exists(server_path):
        register_mcp(skill_dir, ["python3", server_path])
```

The MCP is registered automatically when `/memory-update` runs. The agent sees the skill's tools naturally — no separate install step.

### First Skill MCP: `consistency-check`

Provides a `check_map` tool that parses lookup tables from JSX/TS files:

```
Input:  directory, expected_keys
Output: list of {file, missing_keys}
```

This replaces the agent having to manually grep through 10+ files — the tool does it in milliseconds with zero parsing errors.

## Phase 5: Background Deploy Validation

### What
An AGENTS.md workflow reference telling the agent to run deploy checks in parallel with code changes, using the existing `task-runner` subagent.

### AGENTS.md Addition

```markdown
## Workflow
When starting work on a deploy-bound feature:
1. Kick off `/skill deploy-validation` as a background `task-runner`
2. Work on the fix
3. When the background check finishes, review warnings before deploying
```

| **P1** | `/init-deep` into `/memory*` | Medium | Knowledge plugin TypeScript |

## Token Cost

| Item | Tokens | When |
|------|--------|------|
| Wiki sections in AGENTS.md | ~450 | Every session in project |
| `/init-deep` files | ~5000 total | Every session in project |
| LSP MCP config | ~50 | Idle until called |
| Skill MCPs | ~50 each | Idle until /skill |
| **Total new** | **~5500** | Replaces cold-start discovery tax |
